# Workshop Guide

## 90-minute lightweight workshop

```text
0–10 min: Define work domain and outcome.
10–25 min: Map current work architecture.
25–40 min: Identify roles and responsibility units.
40–55 min: Map boundary states and target states.
55–70 min: Identify movement conditions: evidence, control, accountability, evals, telemetry.
70–80 min: Identify old layout traps and work-architecture shifts.
80–90 min: Capture assumptions, contradictions, and owners.
```

## Output

One Human Responsibility Snapshot, top 3 boundary cards, top 3 work architecture shifts, and top validation tasks.

## Facilitation rules

Do not start with demographic personas. Do not debate automation level before defining accountability. Do not let “AI can do this” become “AI should do this.” Capture disagreements as assumptions or contradictions. Preserve evidence labels. End with decisions or validation tasks.
